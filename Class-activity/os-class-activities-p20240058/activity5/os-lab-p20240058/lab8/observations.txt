Explanation:

The final inventory changed between runs because multiple buy_widget processes accessed inventory.txt at the same time. Several processes could read the same inventory value before another process updated the file. As a result, some updates overwrote others, causing inconsistent inventory values.

This is a Time-of-Check to Time-of-Use (TOC-TOU) race condition. The exact outcome depends on operating system scheduling because concurrent processes may execute in a different order each time.
